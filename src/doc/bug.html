Member settings::scan_config_file (const std::string &option, const std::string &default_key)
Since this is an overloaded function, the compiler needs to know the proper version of the function to call (through the second input argument). The type promotion did also by the compiler may or may not induce it to call the wrong one. The only known case so far is for the std::string type: When the std::string default_key argument is given in terms of quote symbols, " ", i.e. const char*, the compiler can convert it to int type (char, size_t, and bool may bahave as int). Due to that, the function called will be the int type one. To avoid this problem, when the second input argument is given by means of quote symbols, it is highly recomended to specify its type. The layout call should looks like "scan_config_file(option, (std::string) my_default_input_key)";.
